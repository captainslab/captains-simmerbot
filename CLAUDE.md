# Claude Code Instructions

When polishing this repo for sharing:

1. Check `git status`, recent commits, and the diff.
2. Sweep for likely secrets before anything public-facing is updated.
3. Improve `README.md`, install/update docs, and release notes first.
4. Keep startup paths portable and avoid hardcoded personal paths.
5. Add or refresh a changelog when the repo reaches a stable checkpoint.
6. Validate with `git diff --check` and the test suite.
7. Commit, tag, and push only after the repo is clean and ready to share.

Keep the copy concise, visual, and installable by a stranger.
